1. 文档索引流程

流程概述

从文档输入到知识图谱构建的完整流程。支持多种文档格式，包括纯文本、DOCX、PDF等。通过外部解析服务（MinerU/Docling）处理复杂格式，使用VLM分析多模态内容，最终构建完整的知识图谱。

3. 查询模式详解

模式对比

LightRAG 提供6种查询模式，每种模式针对不同的查询场景优化。选择合适的模式可以显著提高查询质量和响应速度。

graph TB
    Query["用户查询"] --> ModeSelection{选择查询模式}

    ModeSelection --> M1["naive
纯向量检索"]
    ModeSelection --> M2["local
局部检索"]
    ModeSelection --> M3["global
全局检索"]
    ModeSelection --> M4["hybrid
混合检索"]
    ModeSelection --> M5["mix
混合+向量"]

    M1 --> D1["检索: 文档块向量
适用: 简单问答"]
    M2 --> D2["检索: 实体向量 + 1-hop邻居
适用: 实体相关问题"]
    M3 --> D3["检索: 关系向量 + 实体
适用: 全局关系问题"]
    M4 --> D4["检索: 实体 + 关系向量
适用: 复杂推理"]
    M5 --> D5["检索: 实体 + 关系 + 块向量
适用: 全面检索"]

    classDef modeClass fill:#e1bee7,stroke:#4a148c,stroke-width:2px
    classDef descClass fill:#c5cae9,stroke:#1a237e,stroke-width:2px

    class M1,M2,M3,M4,M5 modeClass
    class D1,D2,D3,D4,D5 descClass
模式 检索对象 适用场景
naive 文档块向量 简单问答、事实性查询
local 实体向量 + 1-hop邻居 实体相关问题、局部关系查询
global 关系向量 + 实体 全局关系问题、跨实体推理
hybrid 实体 + 关系向量 复杂推理、需要局部和全局信息
mix 实体 + 关系 + 块向量 全面检索、最详细的上下文
bypass 无检索 直接对话、不依赖知识库

核心区别

模式 检索对象 关键词类型 搜索策略
naive chunks_vdb 无 纯向量检索文档块
local entities_vdb LL (低层) 搜索实体 → 找相关关系
global relationships_vdb HL (高层) 搜索关系 → 找相关实体
hybrid entities + relationships_vdb LL + HL local + global 合并
mix entities + relationships + chunks_vdb LL + HL hybrid + 向量块检索

详细说明

naive (简单向量检索)

直接对用户查询进行向量检索，在 chunks_vdb 中搜索最相似的文档块。不使用知识图谱，是最简单的检索方式。

local (局部检索 - 实体中心)

流程：

  1. 提取 LL (Low-Level) 关键词
  2. 在 entities_vdb 中搜索相似实体
  3. 获取找到的实体的 1-hop 邻居关系
  4. 返回实体 + 相关关系

适用："这个实体是什么？"、"X有什么属性？"等实体相关问题

global (全局检索 - 关系中心)

流程：

  1. 提取 HL (High-Level) 关键词
  2. 在 relationships_vdb 中搜索相似关系
  3. 获取关系两端的实体
  4. 返回关系 + 相关实体

适用："X和Y什么关系？"、"A如何影响B？"等关系/推理问题

hybrid (混合检索)

流程：

  1. 同时提取 LL 和 HL 关键词
  2. 并行执行 local 和 global 分支
  3. 使用 Round-robin 策略合并结果

适用：复杂推理，需要同时关注实体和关系

mix (混合+向量)

流程：

  1. 执行 hybrid 检索（实体 + 关系）
  2. 额外进行 chunks_vdb 向量检索
  3. 返回最全面的结果

适用：最全面检索，默认推荐模式

6. 文档删除流程

流程概述

删除文档时需要智能处理实体和关系：如果实体仅被该文档引用则完全删除，如果被多个文档引用则仅移除该文档的引用。删除后可选择重建知识图谱以保持数据一致性。

flowchart TD
    Start([开始: 删除文档]) --> GetDocID["获取文档ID"]

    GetDocID --> GetStatus["获取文档状态
DocStatusStorage.get()"]

    GetStatus --> CheckStatus{文档状态?}

    CheckStatus -->|已索引| GetChunks["获取文档关联的所有块
从chunks KV存储"]
    CheckStatus -->|处理中/失败| DeleteStatus["直接删除状态记录"]

    GetChunks --> GetEntityIDs["获取块关联的实体ID"]

    GetEntityIDs --> GetEntitySources["获取每个实体的source_ids"]

    GetEntitySources --> CheckOtherDocs{实体被其他
文档引用?}

    CheckOtherDocs -->|是| RemoveDocRef["从source_ids移除当前文档"]
    CheckOtherDocs -->|否| MarkDelete["标记实体待删除"]

    RemoveDocRef --> CheckNextEntity
    MarkDelete --> CheckNextEntity

    CheckNextEntity{还有实体?} -->|是| GetEntitySources
    CheckNextEntity -->|否| DeleteMarked["删除标记的实体
从向量存储和图存储"]

    DeleteMarked --> Regraph["重新构建知识图谱
rebuild_knowledge_from_chunks()"]

    Regraph --> DeleteStatus["删除文档状态记录"]

    DeleteStatus --> End([完成])

7. 关键处理阶段说明
